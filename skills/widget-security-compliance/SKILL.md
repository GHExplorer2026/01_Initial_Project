# Widget Security and Compliance

## Use
Use when reviewing source usage, secret handling, logging boundaries, and release evidence.

## Rules
1. No plaintext secrets.
2. Sources must follow approved governance.
3. No scraping bypass behavior.
4. Logs must exclude sensitive payloads.

## Inputs
- Source registry and policy docs
- Runtime config and logging design

## Outputs
- Security checklist
- Compliance checklist
- Verification evidence requirements
